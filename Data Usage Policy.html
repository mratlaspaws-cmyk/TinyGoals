Data Usage Policy for TinyWins

This Data Usage Policy provides detailed information about how TinyWins collects, processes, stores, and uses data to provide goal tracking functionality while protecting your privacy.

1. Types of Data Collected and Used

1.1 Essential App Data Stored Locally

Required for core goal tracking functionality:

Data Type Purpose Storage Location Retention
Goal Information Track your personal goals and progress Local device only Until manually deleted
Progress Data Calculate completion percentages and statistics Local device only Until manually deleted
Statistics Show success rates, streaks, and achievements Local device only Until manually deleted
App Settings Remember your preferences and configurations Local device only Until app uninstalled
Notification State Manage progress alerts and reminders Local device only Until goal completed

1.2 Optional Enhancement Data Requires Permission

Features that require user consent and can be disabled:

  • Notification Permission: Send progress updates and motivational messages
  • Exact Alarm Permission: Precise timing for milestone notifications (Android 12+)
  • Boot Receiver Permission: Restart notifications after device reboot
  • Wake Lock Permission: Keep notifications active during device sleep

1.3 User-Generated Content Your Control

Content you create and fully control:

  • Goal Names: Custom titles for your personal goals
  • Duration Settings: Time periods for goal completion
  • Money Tracking: Optional cost calculations and savings estimates
  • Currency Preferences: Local currency selection for money features
  • Filter Settings: Goal display and organization preferences

1.4 Technical Data Anonymous

Non-personal technical information for app improvement:

  • Device Model: For compatibility testing and optimization
  • Android Version: For feature support and bug fixes
  • App Version: For update management and support
  • Crash Reports: Anonymous technical error information (via AdMob)

2. How Data Is Processed

2.1 Local Processing Architecture

TinyWins Data Flow:

  1. Input: You create goals and track progress
  2. Processing: Calculations happen locally on your device
  3. Storage: Data saved in secure local storage
  4. Display: Results shown in app interface
  5. Notifications: Local alerts triggered by device system

No Step Involves External Servers for your personal goal data.

2.1.1 Goal Tracking Processing

All goal-related calculations are performed locally:

  • Progress percentage calculations
  • Time remaining computations
  • Streak and statistics generation
  • Money savings estimations
  • Motivational message selection

2.1.2 Notification Processing

Notification scheduling uses Android's built-in systems:

  • AlarmManager for precise timing
  • NotificationManager for display
  • Local calculation of milestone triggers
  • Device-based notification delivery

2.2 External Service Processing Third-Party

Limited data processing with external services:

2.2.1 Google AdMob Processing

Note: AdMob processing is separate from TinyWins' core functionality and follows Google's privacy policies.

AdMob may process:

  • Device advertising identifiers
  • App usage patterns (anonymous)
  • Ad interaction data
  • Device technical specifications
  • Approximate location (for relevant ads)

3. Data Storage and Security

3.1 Local Storage Implementation

TinyWins uses Android's secure storage mechanisms:

  • SharedPreferences: Encrypted app settings and preferences
  • JSON Files: Goal data and history in structured format
  • Notification Cache: Temporary data for managing alerts
  • App Sandbox: Isolated storage protecting data from other apps

3.2 Security Measures

Your data is protected by multiple security layers:

  • Device-Level Encryption: Uses Android's built-in encryption when available
  • App Sandboxing: Data isolated from other applications
  • No Network Transmission: Personal goal data never leaves your device
  • Permission Controls: You control access to device features
  • Regular Updates: Security patches delivered through app updates

4. Data Retention and Lifecycle

4.1 Local Data Retention

Your goal data is retained according to your preferences:

Data Category Retention Period User Control Deletion Method
Active Goals Until completion or manual deletion Full control Individual goal deletion
Goal History Indefinitely (local storage) Full control "Clear All Data" feature
Statistics Calculated from goal history Full control Cleared with goal history
App Settings Until manually reset Full control Reset in settings or app uninstall
Notification State Until goal completion Automatic Auto-cleared when goals complete

4.2 Data Backup and Recovery

Important: TinyWins stores data locally. You are responsible for backing up your goal data if desired.

Data backup considerations:

  • No Automatic Backup: Data is not automatically backed up to cloud services
  • Device Transfer: Data won't transfer when you get a new device
  • App Updates: Data persists through normal app updates
  • Manual Export: No built-in export feature currently available

5. Your Control Over Data

5.1 Data Management Tools

TinyWins provides several tools for managing your data:

In-App Data Controls:

  • Individual Goal Deletion: Remove specific goals and their data
  • Clear All Data: Complete reset of all goal history and statistics
  • Notification Settings: Control when and how you receive updates
  • Currency Settings: Change display preferences at any time
  • Filter Management: Control how goals are displayed and organized

System-Level Controls:

  • App Permissions: Enable/disable notifications, alarms, and network access
  • Clear App Data: Android system option to delete all app data
  • App Uninstall: Complete removal of all TinyWins data

6. Data Usage for Specific Features

6.1 Goal Progress Tracking

Uses local data to:

  • Calculate time elapsed since goal start
  • Determine completion percentage
  • Trigger milestone notifications
  • Generate motivational messages
  • Update progress visualizations

6.2 Statistics and Analytics

Processes historical data to show:

  • Success rates across all goals
  • Longest and current streaks
  • Total time committed to goals
  • Money saved through habit changes
  • Average goal completion percentages

6.3 Notification System

Uses goal data to:

  • Schedule milestone alerts (10%, 25%, 50%, 75%, 90%, 100%)
  • Calculate precise notification timing
  • Customize motivational messages
  • Include progress summaries in notifications
  • Provide completion celebration alerts

6.4 Money Tracking Feature

When enabled, processes:

  • User-entered cost amounts
  • Goal duration and progress
  • Currency conversion for display
  • Savings calculations based on completion
  • Historical savings totals

7. Debug and Development Data

7.1 Debug Features

Development Feature: Debug mode is intended for testing and may affect data accuracy.

TinyWins includes debug features that:

  • Allow time acceleration for testing purposes
  • Modify goal timing calculations
  • Generate test data for development
  • Are not intended for production use
  • May affect the accuracy of progress tracking

7.2 Crash Reporting

Anonymous crash data helps improve app stability:

  • Technical Information: Error logs and stack traces
  • Device Context: Android version and device model
  • No Personal Data: Goal information is not included in crash reports
  • Automatic Collection: Processed through AdMob infrastructure

8. Third-Party Service Integration

8.1 Google AdMob Integration

Data shared with AdMob includes:

  • Anonymous device identifiers
  • App usage patterns (no personal goals)
  • Technical device information
  • Ad interaction data
  • Approximate location (if location services enabled)

8.2 Android System Services

Integration with Android system features:

  • Notification Manager: Display progress alerts
  • Alarm Manager: Schedule precise timing events
  • Boot Receiver: Restart notifications after device reboot
  • Shared Preferences: Store app settings securely

9. Data Minimization Principles

9.1 Collection Minimization

TinyWins follows data minimization principles:

  • Purpose Limitation: Only collect data necessary for goal tracking
  • Local Processing: Perform calculations on device when possible
  • Optional Features: Request permissions only when features are used
  • No Unnecessary Data: Don't collect personal information beyond goals

9.2 Processing Minimization

Efficient data processing practices:

  • Process data only when app is actively used
  • Cache calculations to reduce repeated processing
  • Clean up temporary data automatically
  • Minimize background processing
